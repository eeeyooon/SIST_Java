(1) 메모장 
   -코드 작성
   -산출물(결과물) > "Hello.java" > Source Code File
   -소스 파일은 프로그램이 아니다. > 텍스트 파일이다. 
   -프로그래밍 코드를 작성 > 프로그램 완성 > CPU에게 명령어 전달 목적
   -어떤 언어로 구성? > 자바(x) > 사람이 쓰는 언어로 구성
   -소스 파일의 명령어의 실행 주체 > 컴퓨터(CPU) > 이진 데이터 (1, 0)
   
   
(2) javac.exe Hello.java
   -javac.exe > Java Compiler > 사람이 작성한 코드를 컴퓨터가 이해하는 코도로 변역하는 프로그램
   -번역 작업 실행 > Compile
   -산출물 > "Hello.class" > 실행 파일(클래스 파일)
   
(3) java.exe Hello
   -2차 컴파일, 인터프리터, JIT(Just In Time,지트) 컴파일러
   -java.exe > Java Compiler, Java Interpreter, 실행기
   -java.exe 클래스명 > 최종 실행
   -최종적으로 운영체제에 적합한 기계어로 바꿔주는 번역 작업
   
   
 소스 -> 번역 -> 기계어
 1) 컴파일 > 컴파일러 (번역기) > 항상 산출물 발생
 	-장점 : 번역을 한번만 하면 끝 (번역 비용 1회, 하지만 그 1회의 값이 큼)
 	-단점 : 시간이 오래 걸림
 	-초기 실행 비용 비쌈 + 반복 실행 쌈
 		> 자바, SQL
 	
 2) 인터프리트 > 인터프리터 (통역가?) > 산출물 없음
 	-장점 : 시간이 안 걸림 (즉각적으로 번역해주니까) / 비용 발생이 적음
 	-단점 : 읽을 때마다 번역을 해야 함.
 	- 초기 실행 비용 중간 + 반복 실행 비용 중간
 	-수정이 잦은 프로그램은 인터프리터 사용
 		> SQL, HTML, CSS, JavaScript
